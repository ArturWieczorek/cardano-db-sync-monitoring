# Development dependencies (tests, linting, type-checking, pre-commit).
#
# This is intentionally a thin include of requirements.lock, which is the single
# fully-pinned dev set (runtime + dev tools + their transitives), generated from
# pyproject's [dev] extra:
#
#     uv pip compile --python-version 3.10 pyproject.toml --extra dev -o requirements.lock
#
# (--python-version 3.10 = the project floor / CI matrix target, so the lock
#  never pins a package that needs a newer Python, e.g. numpy>=2.3 needs >=3.11.)
#
# Keeping one pinned source avoids drift: a previous hand-maintained list here
# fell behind the lock and the pre-commit hook (older ruff/mypy, missing type
# stubs), so `mypy`/`ruff` in a dev venv disagreed with the gate. Install with:
#
#     uv pip install -r requirements-dev.txt   (or: pip install '.[dev]')

-r requirements.lock
